## 二、设计（论文）进展情况（可直接粘贴中期检查表）

### 1）课题工作量是否适中
整体工作量**适中偏上**，符合本科毕业设计“完整业务闭环 + 工程实现 + 基本测试验证”的要求。

- 已实现“三端 + 风险分析 + 数据库 + 测试”的完整链路，覆盖登录认证、任务发布、学生作答、风险计算、看板展示等核心环节。  
- 技术栈包含前后端分离、权限控制、特征工程、模型训练、可视化与自动化测试，工作内容具有一定综合性与挑战度。

### 2）已完成任务

#### A. 系统架构与基础能力
- 完成前后端分离架构搭建：前端 Vue、后端 Spring Boot、数据库 MySQL。  
- 完成统一接口风格（`ApiResponse`）和跨域/异常处理等基础设施。  
- 完成 JWT + 角色权限控制（学生/教师/管理员）与路由守卫，保障三端访问隔离。

#### B. 后端核心业务模块
- 完成认证模块：`/api/v1/auth/login` 登录并返回 token、用户角色。  
- 完成学生模块：个人信息、任务列表、作业/考试答题提交、视频学习记录、高风险连续天数查询。  
- 完成教师模块：任务发布（作业/考试自动抽题绑定）、活动汇总、风险计算、教师看板。  
- 完成管理员模块：用户分页管理（增删改查）、全局风险看板。

#### C. 风险分析模块（课题特色）
- 完成行为特征构建：成绩、考试失败率、作业缺交率、视频学习、登录活跃度等。  
- 完成逻辑回归式训练流程：历史样本构建、标准化、梯度下降训练、阈值搜索、指标评估（precision/recall/f1/accuracy）。  
- 完成风险结果落库与等级回写（LOW/MEDIUM/HIGH），支持看板展示与学生侧反馈。

#### D. 数据层与测试验证
- 完成数据库结构设计：用户、课程、任务、题库、提交记录、日行为、风险记录等关键表。  
- 完成基础种子数据脚本（50 名学生等），支持联调和演示。  
- 完成后端集成测试（认证流程、教师风险计算）与前端 Cypress 登录 E2E 测试。

### 3）未完成任务（控制为 2 项）
1. **干预策略自动推荐尚未完成**：目前系统能识别风险，但对“高风险学生应采取何种分层干预方案”仍以人工经验为主。  
2. **预警解释性前端展示不够完整**：后端已有 `detail_json` 特征细节，但学生端/教师端尚未形成“特征贡献 + 可视化解释”的完整交互页面。

### 4）完成课题存在问题与解决办法

#### 问题 1：历史样本质量与规模对模型效果影响较大
- **表现**：样本不足时会回退默认参数，模型指标波动较大。  
- **解决办法**：
  - 扩充历史行为数据周期（例如 8~16 周）；
  - 完善缺失值处理与异常值抑制；
  - 增加离线评估集，固定版本对比 precision/recall/f1。

#### 问题 2：当前预警结果“可用”，但“可解释、可干预”仍可加强
- **表现**：系统能给风险等级，但干预建议模板化，个性化不足。  
- **解决办法**：
  - 基于风险特征主因生成干预建议（如“低作业完成+低登录”→督学提醒）；
  - 在前端增加“风险构成条形图/雷达图”；
  - 建立“干预后复测”闭环，观察风险变化趋势。

**讲稿提示**：
“系统不是单点功能，而是完整业务闭环，重点在风险识别与角色化呈现。”

**讲稿提示**：

- 配套计划：补充历史数据、做离线指标对比、迭代阈值策略

**讲稿提示**：
“后续重点是把预警结果变成可执行的教学决策支持。”
