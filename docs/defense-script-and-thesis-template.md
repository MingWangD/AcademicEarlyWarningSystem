# 答辩演示逐步台词稿（学业预警系统）

> 建议总时长：6~8 分钟。可按“系统目标 → 三端演示 → 预警闭环 → 总结展望”展开。

## 1. 开场与研究目标（约 30 秒）
“各位老师好，我的课题是《学业预警系统的设计与实现》。本系统面向学生、教师与管理员三类角色，目标是通过学习行为与成绩数据的采集、分析与可视化，实现学业风险的早发现、早预警和早干预。”

## 2. 系统总体架构说明（约 40 秒）
“系统采用前后端分离架构：前端使用 Vue 构建三端页面，后端使用 Spring Boot 提供 REST 接口，数据库存储用户、任务、题库、作答记录与风险结果。业务主线是：教师发布任务 → 学生完成任务 → 系统汇总行为特征 → 风险模型计算 → 看板展示预警结果。”

## 3. 学生端演示（约 2 分钟）

### 3.1 登录与任务列表
“现在进入学生端。学生登录后可以看到当前课程下的任务列表，包括作业、考试和学习任务。每条任务都有状态与截止时间，便于学生安排学习节奏。”

### 3.2 作业/考试答题
“我选择一条作业任务进入答题页。系统会根据课程与任务类型自动加载题目。学生完成作答后提交，系统立即返回成绩，并将作答结果写入提交记录。”

### 3.3 学生预警信息查看
“学生还可以在预警页面查看个人风险等级与关键指标变化，了解自己在哪些行为维度需要改进，例如作业完成率、考试成绩或学习活跃度。”

## 4. 教师端演示（约 2 分钟）

### 4.1 发布任务
“切换到教师端。教师在发布任务时选择课程和任务类型，系统可从题库按规则自动组题并绑定任务，减少重复出题的工作量。”

### 4.2 行为汇总与风险计算
“教师可查看班级行为汇总，如提交率、平均分、活跃度等。点击风险计算后，系统根据多维特征输出每位学生的风险分和等级（高/中/低），辅助教师识别重点关注对象。”

### 4.3 教学干预价值
“通过该结果，教师可以更有针对性地进行辅导，例如对高风险学生安排答疑、督促补交或个别沟通，形成数据驱动的教学干预。”

## 5. 管理员端演示（约 1 分钟）
“管理员端主要负责全局管理，包括用户管理与全校风险看板。管理员可以从宏观层面查看风险分布和趋势，为教学管理与资源配置提供依据。”

## 6. 闭环总结（约 40 秒）
“系统形成了完整闭环：任务发布、学生完成、数据沉淀、风险识别和结果反馈。相较于传统依赖人工经验的方式，本系统具备更高的实时性与可量化能力，能够更早发现潜在学业问题。”

## 7. 结束语（约 20 秒）
“以上是本系统的演示。后续将进一步优化模型精度与干预策略推荐机制，提升预警的可解释性与实用价值。谢谢各位老师。”

# “论文实现章节模板”（可直接替换细节）

> 章节命名建议：第 4 章 系统实现。

## 4.1 开发环境与技术栈
- 前端：Vue + Vue Router + Axios + Element Plus
- 后端：Spring Boot + MyBatis（或 JPA）
- 数据库：MySQL
- 认证授权：JWT + RBAC（学生/教师/管理员）
- 开发与部署工具：Maven、Node.js、Nginx（如有）

> 可补充一张“技术栈总览表”。

## 4.2 系统总体实现
### 4.2.1 架构实现
- 描述前后端分离架构与模块边界
- 描述接口调用流程与数据流向

### 4.2.2 角色与权限实现
- 三端入口与路由隔离
- 接口权限校验机制

## 4.3 核心功能实现

### 4.3.1 登录认证模块
- 登录请求与响应结构
- JWT 签发、存储与拦截器校验流程
- 异常处理（未登录、令牌过期、权限不足）

### 4.3.2 教师任务发布模块
- 输入项：课程 ID、任务类型、截止时间、说明
- 核心逻辑：按课程与任务类型从题库抽题并绑定任务
- 输出：任务发布结果与任务标识

### 4.3.3 学生任务完成模块
- 学生任务列表查询
- 作业/考试页面拉取题目与答题提交流程
- 自动判分与提交记录持久化

### 4.3.4 行为数据汇总模块
- 汇总维度：提交次数、成绩、活跃度、学习时长等
- 汇总策略：按日/按周聚合
- 为风险分析模块提供标准化特征

### 4.3.5 风险分析与预警模块
- 特征构建方法
- 风险评分计算（可写逻辑回归/规则融合）
- 风险等级划分阈值与预警结果存储

### 4.3.6 可视化看板模块
- 学生端：个人风险与学习画像
- 教师端：班级风险分布、重点学生列表
- 管理员端：全局统计与趋势图

## 4.4 数据库设计与实现
### 4.4.1 概念结构设计
- 给出 ER 图并说明关键实体关系

### 4.4.2 关键数据表设计
建议重点介绍以下表（按实际项目替换）：
- user（用户）
- course（课程）
- task（任务）
- question_bank（题库）
- task_question（任务-题目关联）
- homework_submission / exam_submission（提交记录）
- student_daily_activity（行为聚合）
- risk_record（风险结果）

> 每张关键表建议包含：字段说明、主外键、索引设计、用途说明。
